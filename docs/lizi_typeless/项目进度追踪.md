## 已确认决策

1. 项目仅供个人使用，以核心语音输入体验为目标，不按商业产品建设。
2. Windows 11 运行 WPF 客户端，WSL2 运行本地 CUDA 推理服务。
3. 空闲时 350 ms 内双击右 Alt 开始录音，录音时再次按下右 Alt 结束，结束后抑制 650 ms 内的重复按键。
4. 录音期间只显示流式预览，最终文本在结束后一次性写入。
5. 智能整理允许去口头语、标点分段、明确改口和自然列表，但忠实原意优先。
6. 原始音频持续落盘，处理失败必须可从历史记录 Retry；Retry 不自动写回旧目标。
7. 推理服务只监听回环地址，运行期不把语音或文本发送到外部 API。
8. 当前 ASR 候选为 Qwen3-ASR-1.7B/vLLM eager，整理候选为 Qwen3-1.7B/Transformers；50 条个人语音集通过前不视为最终模型结论。
9. Windows 与 Python HTTP JSON 统一使用 `snake_case`，WebSocket 控制消息继续使用当前 camelCase 合同。
10. WPF 必须发布到 Windows 本地目录运行，不从 WSL UNC 输出目录直接启动。
11. 项目对内对外统一使用 `lizi_typeless`，包括仓库目录、解决方案、.NET 标识、Python 包、环境变量、UI、文档、发布目录和活动数据目录。

## 风险与处理

1. **个人语音样本不足**：改名前首批 10 条经当时的精确个人词表后达到 CER 和技术词目标，但还需 40 条覆盖更多真实表达，并补充 `lizi_typeless` 的真实口述；不能从小样本直接确定模型。
2. **长文本整理可能超时**：50.45 秒合成长样本的转录收尾约 99 ms；重复整段已用窄规则把整理降到约 197 ms，但早期 154 字生成耗时约 1131 ms，普通长文本仍需个人样本确认。
3. **录音启动与首字完整性样本不足**：修复版单次录音在 98.8 ms 就绪，首帧回调 230.3 ms且未丢首字；仍需 30 次 P95 判断稳定性。
4. **跨语言合同漂移**：首条实机样本暴露 `Text`/`text` 造成 422；已统一命名策略并加入 Windows 合同测试。
5. **文本误投或重复**：开始时保存目标窗口，结束时复核窗口句柄和进程；目标变化时停止自动写入，Retry 永不自动插入。
6. **推理或客户端异常导致内容丢失**：Windows 先持续落盘，流式失败回退完整 WAV，启动时恢复未完成会话。
7. **模型常驻资源已量化**：两组 10 分钟采样显示功耗中位数增量 7.70 W、平均增量 8.93 W，GPU 利用率中位数和 P95 增量均为 0；量化目标通过，风扇主观表现继续随真实使用观察。
8. **改名期间历史数据丢失**：11 条有效会话复制到新目录后完成 33 个文件的逐一 SHA-256 对比，旧目录继续保留；新客户端只写新目录。
