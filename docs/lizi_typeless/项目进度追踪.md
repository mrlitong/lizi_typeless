## 已确认决策

1. 项目仅供个人使用，以核心语音输入体验为目标，不按商业产品建设。
2. Windows 11 运行 WPF 客户端，WSL2 运行本地 CUDA 推理服务。
3. 空闲时 350 ms 内双击右 Alt 开始录音，录音时再次按下右 Alt 结束，结束后抑制 650 ms 内的重复按键。
4. 录音期间只显示流式预览，最终文本在结束后一次性写入。
5. 智能整理允许去口头语、标点分段、明确改口和自然列表，但忠实原意优先。
6. 原始音频持续落盘，处理失败必须可从历史记录 Retry；Retry 不自动写回旧目标。
7. 推理服务只监听回环地址，运行期不把语音或文本发送到外部 API。
8. 当前采用 Qwen3-ASR-1.7B/vLLM eager 与 Qwen3-1.7B/Transformers 作为个人 MVP 模型组合，后续随真实使用继续优化。
9. Windows 与 Python HTTP JSON 统一使用 `snake_case`，WebSocket 控制消息继续使用当前 camelCase 合同。
10. WPF 必须发布到 Windows 本地目录运行，不从 WSL UNC 输出目录直接启动。
11. 项目对内对外统一使用 `lizi_typeless`，包括仓库目录、解决方案、.NET 标识、Python 包、环境变量、UI、文档、发布目录和活动数据目录。

3. **录音启动与首字完整性继续观察**：现有真实录音没有发现首字丢失；不设专项次数门槛，后续在个人使用中保留并反馈异常 case。

6. **推理或客户端异常导致内容丢失**：Windows 先持续落盘，流式失败回退完整 WAV，启动时恢复未完成会话。
