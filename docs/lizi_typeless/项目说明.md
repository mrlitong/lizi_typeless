# lizi_typeless 项目说明

> 状态：已定（个人 MVP 已验收）
> 更新日期：2026-07-11（Asia/Shanghai）

## 一句话定义

lizi_typeless 是一个面向个人使用的 Windows 11 本地语音输入工具，通过全局快捷键录音、本地语音识别和忠实的智能整理，将普通话及中英文技术内容一次性写入当前应用。

## 目标与价值

1. 把日常口述快速转成可直接发送或继续编辑的文字。
2. 优先保证原意准确、响应迅速和长短录音均不丢失。
3. 使用 RTX 5090 在本地完成推理，避免网络波动、云端限流和外部 API 依赖。
4. 为个人开发工作优化普通话、英文产品名、编程术语、命令和技术内容的识别体验。

## 当前范围

1. Windows 11 宿主机运行桌面客户端，WSL2 运行适合 Linux/CUDA 的本地推理服务。
2. 空闲时双击右 Alt 开始录音，录音时再次按下右 Alt 结束。
3. 录音期间显示状态和实时预览，结束后完成最终转录、智能整理并一次性写入目标输入框。
4. 主要支持浏览器、微信、VS Code 和 Windows Terminal。
5. 持续保存原始录音，并保留原始转录、整理结果、历史记录和手动重试能力。
6. 模型冷启动后常驻；每次录音立即响应，空闲时不主动执行推理。

## 非目标

1. 商业发布、账号系统、遥测、自动更新和面向多用户的配置体系。
2. 正式 Windows TSF 输入法、系统语言栏集成和全应用兼容。
3. 游戏、全屏应用、密码框、受保护输入框和管理员权限窗口的优先适配。
4. 上传语音或文本到外部 API。
5. 自动清理历史数据、云端同步和跨设备同步。
6. 在 MVP 阶段训练或微调语音模型。

## 关键决策

1. 录音期间使用流式识别生成预览并降低尾延迟，但临时文本不写入目标输入框。
2. 用户结束录音后，最终文本一次性写入，避免输入框内反复改写。
3. 智能整理以忠实复述为最高优先级；排版美观不得覆盖原意准确。
4. 原始音频从录音开始持续落盘；推理或写入失败不得使已经录制的内容消失。
5. Windows 客户端采用 .NET 10 WPF；推理服务采用 FastAPI、本机 WebSocket/HTTP、Qwen3-ASR-1.7B 和 Qwen3-1.7B。
6. 当前 Qwen3-ASR-1.7B/vLLM eager 与 Qwen3-1.7B/Transformers 组合已作为个人 MVP 采用方案；后续继续随真实使用扩充个人样本。
7. 项目对内对外统一使用 `lizi_typeless`；仓库、解决方案、程序集、命名空间、Python 包、环境变量、窗口标题和本地数据目录不保留其他项目标识。

## 重要约束

1. 主要语言是普通话，内容经常包含 Docker、K8s、Claude Code、Codex、Azure 等英文产品名和开发术语。
2. 单次录音通常较短，最长约一分钟、几百字。
3. RTX 5090 可由本项目独占，但模型常驻后应保持低空闲功耗和无明显风扇噪音。
4. 原始录音和历史文本默认永久保留，由用户手动清理。
5. 项目追求个人自用的最小可靠实现，不为理论扩展性增加复杂度。
6. 当前活动数据目录是 `%LOCALAPPDATA%\lizi_typeless`；改名前的数据副本暂时保留，只作为人工回退备份。

## 当前状态

个人 MVP 核心验收已经完成。Windows 客户端、流式推理、历史、恢复、Retry 和一次性写入主链路可用；个人固定集 20/20、3 条长录音、自动化测试和空闲资源目标通过。首轮完整报告暴露第 16、17、20 条数字事实变更和否定限制遗漏；当前已增加忠实度保护，定向重放全部保留正确原始信息，修复后完整重放没有出现新的核心语义回退。原始 CER、技术词格式、项目名识别和跨应用焦点继续作为个人使用中的后续优化，不阻断当前 MVP。
